---
name: NWScript / KotOR conventions
description: Guidelines when editing NWScript (.nss) or KotOR mod files
---

When editing NWScript or KotOR modding files in this workspace:

- Use KotOR/TSL built-in function names (e.g. GetPC, GetFirstObjectInArea, ApplyEffectToObject).
- Constants and effect/visual IDs follow the game's nwscript.nss and script library.
- Prefer C-style braces and indentation; main() is the usual entry point for scripts.
- For GFF/2DA/TLK and other binary formats, use the KotOR Forge editors or JSON view where applicable.
